# Глава 4. Audit text export

Статус: текстовый экспорт для автоматической проверки до фиксации финального DOCX.

Ключевой результат главы 4: FuzzyXAI описывается как программно-методическая экосистема включения внешних объяснительных артефактов в проверяемый маршрут операторов. Пользовательский слой не является отдельной теорией и не подменяет вычислительное ядро.

Инварианты маршрута:
- каждый объект, влияющий на действие, имеет трассируемый источник;
- редукция представления неопределённости сопровождается вычислением Δ;
- автоматическое действие формируется только после риск-ориентированного наблюдателя;
- невозможность построения или согласования объекта возвращает диагностическое состояние.

Связь с кодом:
- `T_ij`: `fuzzyxai/core/alignment.py`;
- `Δ`: `fuzzyxai/core/reduction.py`;
- `ρ`, `χ_R^{crit}`: `fuzzyxai/core/risk_observer.py`;
- proof package: `fuzzyxai/core/proof_package.py`;
- batch-run: `fuzzyxai/run_scenario.py`;
- экспорт таблиц: `fuzzyxai/export_tables.py`.

Граница ответственности: FuzzyXAI проверяет сохранение объяснительных каналов, диагностические разрывы, редукцию, действие по ExplainPlan и воспроизводимость маршрута; она не заявляет предметную сертификацию исходной модели.
