# FuzzyXAI: единое связное изложение (главы 2–3 + observer + эксперименты)

## Назначение
Этот файл хранит всё в одном месте: постановка задачи, математическая линия, программная реализация и проверка на данных.

## 1) Проблема
Модульные ИИ-конвейеры дают разрозненные объяснения. Нужен формальный контур, который:
- строит единый объяснительный объект на каждом шаге;
- проверяет согласование между шагами;
- композиционно собирает цепочку;
- при несогласовании возвращает диагностический разрыв;
- принимает риск-ориентированное действие.

## 2) Глава 2: системный оператор
Нечёткий объяснительный объект компонента:
- термы `L`;
- представление `A^F`;
- правила `R`;
- активации `alpha`;
- неопределённость `u`;
- trace `tau`.

Композиция строится через согласование `T_ij` и меру `d_E`.
Если `d_E <= gamma_max` — есть композиция, иначе `D_ij`.

Индекс интерпретируемости:
`I(E)=exp(-L(E))`, где `L(E)` учитывает энтропию, сложность правил, перекрытие, конфликтность и неопределённость.

## 3) Глава 3: иерархия представлений неопределённости
Классы: `F0`, `F_int`, `F_H`, `F_N^src`, `F_ML`.

Выбор идёт по профилю ситуации `P_sit`:
- покрыть типы неопределённости;
- выбрать минимально достаточный класс по Парето-критериям.

При редукции вычисляется потеря `Delta`, которая передаётся в риск-слой.

## 4) Category/HoTT слой
Без рискованных утверждений:
- строим категорию успешных согласований `Expl`;
- над ней — предпучковый слой контекстов `Set^{Expl^op}`;
- успешный переход = `Path`, неуспешный = `Rupture`.

## 5) Risk-aware observer
Интегральный риск:
`rho = w_p*rho_p + w_u*u + w_I*(1-I_pre) + w_Delta*Delta + w_R*chi_R`.

Действия:
`accept`, `lower_confidence`, `request_more_data`, `defer_to_human`, `block`.

Критический разрыв `chi_R=1` запрещает «тихий» авто-accept.

## 6) Эксперименты (глава 5)
- сценарии `S0..S6`;
- калибровка весов;
- сравнение baseline;
- sensitivity (`w_R`, пороги);
- timing;
- breast cancer validation.

## 7) Что реализовано в коде
Ключевые модули:
- `fuzzyxai/core/` — оператор, композиция, диагностика, trust;
- `fuzzyxai/hierarchy/`, `fuzzyxai/selection/` — иерархия и выбор;
- `fuzzyxai/risk/` — observer и риск;
- `fuzzyxai/category/`, `fuzzyxai/hott/` — категориально-гомотопический слой;
- `experiments/` — воспроизводимые эксперименты и отчёты.

## 8) Единый GUI и единый CLI-демо

FuzzyXAI переводит объяснимость из «комментария после прогноза» в активный контур безопасности: если цепочка объяснения структурно разорвана, система не делает рискованное авто-действие и передаёт решение в безопасный режим.
